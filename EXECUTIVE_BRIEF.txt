CRITICAL FINDINGS (5 Blockers)

1. CRITICAL — Content Injection in PDF Generation
   File: app/pdf.py:170,178,214,223,233
   Risk: User input directly embedded in PDF without escaping; markup injection possible
   Impact: PDF rendering breaks; misleading scores; potential attack vector
   Fix: Escape user text; validate tier values; add tests
   Effort: 3–4 hours

2. CRITICAL — API Key Exposed in .env.example
   File: .env.example, README.md
   Risk: Example contains API key pattern; no clear offline/online setup docs
   Impact: New developers expose real keys; first deployment fails
   Fix: Replace with placeholders; add OS-specific setup guides
   Effort: 1 hour

3. HIGH — Path Traversal in Fixture Loading
   File: app/content.py:33
   Risk: No filename validation; attacker could read secrets.yaml, .env files
   Impact: Unauthorized file access; privilege escalation
   Fix: Validate path stays within FIXTURE_DIR; add unit tests
   Effort: 2–3 hours

4. HIGH — Memory Exhaustion (Unbounded PDF/YAML)
   File: app/pdf.py:251, app/content.py:14
   Risk: No size caps on PDF or YAML files
   Impact: Crafted data → multi-GB PDF → disk full → service down
   Fix: Cap PDF to 50 MB, YAML to 10 MB; early error detection
   Effort: 2–3 hours

5. HIGH — Missing Windows Setup Instructions
   File: README.md
   Risk: .venv/Scripts/activate path wrong for Windows; no OS-specific guidance
   Impact: Windows developers cannot run app; 50% setup failure rate
   Fix: Document Windows PowerShell/Bash, macOS, Linux activation
   Effort: 1 hour

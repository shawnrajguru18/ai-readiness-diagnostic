TIER 1: CRITICAL SECURITY (6 vulnerabilities - 4 blockers)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.1 PROMPT INJECTION - [BLOCKER]
   Location: app/agents/__init__.py:51-52, app/api.py:99-105
   Risk: Attacker can override LLM system instructions via unvalidated user input
   Example: prospect_name = "John\nFORGET YOUR INSTRUCTIONS"
   Impact: Voice constraints bypassed, sensitive data leakage
   Fix Effort: 4-6 hours

1.2 PATH TRAVERSAL - [BLOCKER]
   Location: app/content.py:32-33
   Risk: Load arbitrary files via directory traversal in fixture endpoint
   Example: /api/fixture/../../etc/passwd → accesses app/config.py
   Impact: Access to configuration, secrets, source code
   Fix Effort: 2-4 hours

1.3 HTTP HEADER INJECTION - [BLOCKER]
   Location: app/api.py:170-172, 177-179
   Risk: CRLF injection in PDF Content-Disposition filename
   Example: company_name = "Corp\r\nX-Injected-Header: value"
   Impact: Header injection, response splitting, cache poisoning
   Fix Effort: 1-2 hours

1.4 CORS MISCONFIGURATION
   Location: app/api.py:37
   Risk: Allow requests from any origin (allow_origins=["*"])
   Impact: Cross-site request forgery, data exfiltration
   Example: Attacker site can POST to /api/assess and /api/review/queue
   Fix Effort: 1-2 hours

1.5 EMAIL VALIDATION MISSING
   Location: app/models.py:42
   Risk: Invalid emails accepted, can be used for injection
   Example: "user@example.com\r\nBcc: attacker@evil.com"
   Impact: Email injection, LLM prompt injection vector
   Fix Effort: 2-4 hours

1.6 URL VALIDATION MISSING
   Location: app/models.py:44
   Risk: No validation on company_website field
   Example: "http://localhost:9000/admin", "file:///etc/passwd"
   Impact: SSRF attacks, internal service access
   Fix Effort: 2-4 hours

TIER 2: DATA INTEGRITY (5 vulnerabilities - 2 blockers)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2.1 RACE CONDITION - DYNAMODB - [BLOCKER]
   Location: app/store.py:87-88 (_DynamoStore.put)
   Risk: Unconditional put_item overwrites concurrent writes
   Scenario: Two assessments created simultaneously with same ID → data loss
   Impact: Lost assessments, data integrity violation
   Fix Effort: 8-12 hours (requires optimistic locking)

2.2 RACE CONDITION - REVIEW DECISION - [BLOCKER]
   Location: app/api.py:150-160
   Risk: Read-modify-write without locking
   Scenario: Two reviewers decide simultaneously → last write wins
   Impact: Lost review decision, audit trail incomplete
   Fix Effort: 6-8 hours

2.3 BANKER'S ROUNDING INCONSISTENCY
   Location: app/scoring.py:90, 104
   Risk: Python's round() uses banker's rounding (round half to even)
   Issue: 50.5 rounds to 50 (not 51), inconsistent tier boundaries
   Impact: Incorrect scores near thresholds (39.5→39, 59.5→60)
   Fix Effort: 2-4 hours

2.4 TYPE MISMATCH IN SCALE SCORING
   Location: app/scoring.py:28-34
   Risk: YAML may have score as string or int (inconsistent types)
   Issue: Type coercion masks errors, scoring fails silently
   Impact: Inaccurate dimension scores
   Fix Effort: 2-3 hours

2.5 EVENTUAL CONSISTENCY LAG
   Location: app/store.py:96-106 (scan), app/api.py:109-129 (queue)
   Risk: DynamoDB scan uses eventually consistent reads
   Issue: Newly created assessments may not appear in review queue immediately
   Impact: Partner reviewers don't see all pending reviews
   Fix Effort: 3-5 hours

TIER 3: RESILIENCE & ERROR HANDLING (5 vulnerabilities)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3.1 BARE EXCEPTION CLAUSES
   Location: app/agents/__init__.py:58-59, 115-116
   Risk: "except Exception:" catches all errors including MemoryError
   Issue: Bugs hidden, MemoryError masked, system silently degrades
   Impact: Difficult debugging, unexpected behavior
   Fix Effort: 3-5 hours

3.2 NO RATE LIMIT HANDLING
   Location: app/llm.py:44, 65
   Risk: HTTP 429 (Too Many Requests) not caught or retried
   Issue: When API rate limited, request fails hard with no retry
   Impact: Service degradation under load, no graceful fallback
   Fix Effort: 4-6 hours

3.3 NO TIMEOUT ON LLM CALLS
   Location: app/llm.py:44, 65
   Risk: No timeout configured on LLM API calls
   Issue: Hanging API can block endpoint indefinitely, exhaust connection pool
   Impact: Cascading failures, DOS vulnerability
   Fix Effort: 2-4 hours

3.4 MISSING ERROR HANDLING IN JSON DESERIALIZATION
   Location: app/store.py:45-55 (_rec_from_json)
   Risk: json.JSONDecodeError and ValidationError not caught
   Issue: Corrupted DynamoDB record causes complete failure
   Impact: Can't retrieve assessment if data corrupted
   Fix Effort: 2-3 hours

TIER 4: DATA VALIDATION (1 vulnerability)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4.1 LENGTH CONSTRAINTS MISSING
   Location: app/models.py (Submission, QuestionResponse)
   Risk: No max_length on text fields
   Issue: Extremely long inputs (50KB+) cause:
           - LLM prompt token overflow
           - PDF generation failures
           - Database storage issues
   Impact: DOS vulnerability, data corruption
   Fix Effort: 1-2 hours
